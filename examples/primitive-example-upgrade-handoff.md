# Primitive Example Upgrade Handoff

This document is a follow-up brief for the next agent improving the remaining
primitive examples.

Reference implementations:

- `examples/example-template/` is the canonical bootstrap pattern.
- `examples/borough-choropleth/` is the current strongest example of the report
  split, tracked tearsheet assets, and question-driven chart design.

## Shared Upgrade Rules

- Keep `cache/` ignored for reusable local snapshots.
- Keep `artifacts/` ignored for scratch CSVs, intermediate outputs, and any
  heavy non-report assets.
- Add tracked `reports/` when the example produces stable markdown and stable
  figures that belong in git.
- Use `reports/<example-slug>-tearsheet.md` for the main markdown file.
- Put tracked images under `reports/figures/`.
- Use explicit relative markdown image paths like `./figures/example-chart.png`.
- Prefer auto-computed tearsheet prose over tutorial-style narration inside the
  report.
- Keep README user-facing. Put implementation explanation in code comments or in
  this handoff, not inside the report body.
- Frame each example around a small number of concrete analytical questions.
- For sample-backed examples, tracked reports can be generated by default.
- For cache-backed live examples, tracked reports should usually be gated behind
  a publish step such as `--publish-report` or an explicitly refreshed pinned
  cache.
