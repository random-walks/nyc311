---
description:
  Run the nyc311 release preflight — invokes the release-auditor agent against
  the current branch.
---

Invoke the [`release-auditor`](../agents/release-auditor.md) agent against the
current branch. Relay its report verbatim; do not summarize or editorialize. If
the agent flags a blocking item, print the agent's remediation hint and stop —
do not attempt fixes yourself.

If the agent returns a clean report, also print:

```
Next step:
  git tag vX.Y.Z
  git push --follow-tags
The CD workflow will publish to PyPI via OIDC on the release event.
```

Do NOT tag, do NOT push, do NOT touch `_version.py` (it is generated by
`hatch-vcs`).
